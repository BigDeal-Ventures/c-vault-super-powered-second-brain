# Vault Structure

The starter vault uses a simple PARA-inspired layout with a metadata layer.

## Folders

- `00_Inbox`: raw capture and unprocessed notes.
- `01_Projects`: active outcomes with end states.
- `02_Areas`: ongoing responsibilities and standards.
- `03_Resources`: reusable reference and research.
- `04_Archive`: inactive or completed material.
- `05_Attachments`: files used by notes.
- `06_Metadata`: templates, instructions, command references, and operating guides.
- `07_Clients`: client-specific spaces when needed.

## Naming

- Use clear human-readable names.
- Prefer dates in `YYYY-MM-DD` format.
- Keep client and project folders stable once linked.
- Avoid tool-specific prefixes unless they help sorting.

## Handoffs

Any active project or client folder should include a short handoff section:

- Current state
- Next action
- Open questions
- Important links
- Risks or constraints
